# API 返回格式错误修复

## 问题描述

豆包 API 有时返回单个数字（如 `32`）而不是完整的 JSON 对象，导致评测失败。

## 解决方案

**新增功能：**
- ✅ 自动重试机制（最多2次）
- ✅ 增强 JSON 格式要求的系统提示词
- ✅ 智能检测和处理异常返回格式：
  - 单个数字 → 转换为包含分数的 JSON 对象
  - 单个字符串 → 记录警告并返回默认结构
  - 空列表/非字典类型 → 自动重试或返回错误结构

## 技术细节

### 支持的异常格式处理

| API 返回 | 处理方式 | 结果 |
|---------|---------|------|
| `32` | 转换为 JSON | `{"total_score": 32, "max_score": 100, ...}` |
| `"some text"` | 记录警告，返回默认分 | `{"total_score": 0, "max_score": 100, ...}` |
| `[{"score": 80}]` | 提取第一个元素 | `{"score": 80}` |
| `[]` | 重试或返回错误 | `{"error": "API 返回了空列表"}` |

### 重试策略

- **最大重试次数**: 2次
- **重试间隔**: 1秒
- **重试触发条件**:
  - 返回非字典类型
  - JSON 解析失败
  - 返回单个数字或字符串
